## 🚨 重要规则与流程

### 规则1: 方案必须先讨论后执行
- ❌ **禁止**: 未经讨论直接执行方案
- ✅ **必须**: 所有方案必须先提出，等待确认后才能执行
- 📋 **流程**: 提出方案 → 数据支撑 → 讨论修改 → 确认 → 执行

### 规则2: 基于数据的决策
- 所有方案必须有数据支撑
- 需要分析小红书真实案例
- 需要参考竞品内容表现
- 结论必须有依据可循

### 规则3: 文档化管理
- 每个可行方案输出独立的markdown文档
- 文档命名格式: `方案_{类型}_{日期}.md`
- 保存路径: `xiaohongshu_automation/plans/`

### 规则4: 内容合规
- 避免绝对化用语（最好、第一）
- 不夸大宣传效果
- 不做虚假承诺
- 符合小红书平台规范

## 📊 工作流程

### 阶段1: 数据收集与分析
```
1. 搜索小红书相关内容
   - 关键词: "Temu赚钱"、"Temu培训"、"跨境电商无货源"
   - 分析高互动内容
   - 提取成功模式

2. 竞品分析
   - 找到类似产品/服务的营销内容
   - 分析标题、文案、话题标签
   - 记录点赞、收藏、评论数据

3. 输出分析报告
   - 创建 `analysis_report_{date}.md`
   - 包含数据表格和结论
```

### 阶段2: 方案设计
```
1. 基于数据提出初步方案
   - 内容方向
   - 标题模板
   - 文案结构
   - 话题标签策略

2. 讨论与优化
   - 提交方案给用户
   - 收集反馈
   - 修改完善

3. 确认方案
   - 用户明确确认后才能继续
```

### 阶段3: 内容生成
```
1. AI生成内容
   - 使用确认的模板
   - 生成标题+文案+标签

2. 图片/视频方案
   - 提供生成方案（AI工具、设计工具）
   - 提供提示词或设计建议

3. 审核机制
   - 生成内容先保存为草稿
   - 用户审核后再发布
```

### 阶段4: 发布与追踪
```
1. 使用小红书MCP发布
2. 记录发布日志
3. 追踪互动数据
4. 定期复盘优化
```

## 🎨 内容生成规范

### 标题规范
- 字数: 12-20字
- 必须包含: 数字/痛点/利益点之一
- 避免: 绝对化用语
- 格式: 参考高互动案例

### 正文规范
- 字数: 200-400字
- 结构: 痛点引入 → 解决方案 → 效果展示 → 行动召唤
- 风格: 口语化、真诚、有共鸣
- Emoji: 适度使用，增强可读性

### 话题标签规范
- 数量: 6-8个
- 组合: 1个大流量 + 2-3个精准 + 2-3个相关
- 避免: 堆砌无关标签

### 图片规范
- 尺寸: 1242x1656 px（小红书推荐比例）
- 数量: 3-9张
- 内容: 清晰、专业、有信息量
- 风格: 统一色调和设计语言

### 视频规范
- 时长: 15-60秒
- 开头: 3秒内抓住注意力
- 结构: 钩子 → 痛点 → 解决方案 → CTA
- 字幕: 必须有清晰字幕

## ⚠️ 风险管理

### 内容风险
- **风险**: 违反平台规则，账号被限流或封禁
- **应对**: 严格遵守内容规范，避免敏感词汇

### 竞争风险
- **风险**: 竞品抢占流量和用户
- **应对**: 持续创新内容，突出差异化价值

### 效果风险
- **风险**: 投入大量精力但转化效果不佳
- **应对**: 小步快跑，快速测试和迭代

### 技术风险
- **风险**: 自动化系统故障或MCP接口变更
- **应对**: 保留人工审核机制，建立应急预案

## 📞 沟通与确认机制
